title: "Generated Files Reference"
description: "Reference for the generated state files agentpack writes under .agentpack/."
---

agentpack manages several state files under the `.agentpack/` directory. Some are meant to be committed, others are runtime-only.

## Summary

| File | Produced by | Commit? | Purpose |
|---|---|---|---|
| `compiled.json` | `author build`, `publish validate` | **Yes** | Semantic source of truth: compiled skills, source hashes, graph edges |
| `install.json` | `materialize`, `skills enable`, `skills disable` | No | Direct-package/runtime selection state for consumer materialization |
| `materialization-state.json` | `materialize`, `skills enable`, `skills disable`, `author dev` | No | Runtime output ownership: which adapter owns which materialized files |
| `dev-session.json` | `author dev` | No | Active dev session PID and linked skill paths |

---

## `.agentpack/compiled.json`

The semantic source of truth for the skill graph. Contains compiled skills, source content hashes, body reference occurrences, and aggregated graph edges.

**Produced by:** `agentpack author build <target>`, `agentpack publish validate <target>`

**Structure:**

```json
{
  "rootSkill": "brand-copywriting",
  "skills": {
    "brand-copywriting": {
      "name": "brand-copywriting",
      "description": "Write brand-consistent copy...",
      "skillImports": { ... },
      "sourceBindings": { ... },
      "occurrences": [ ... ],
      "edges": [
        {
          "source": "skill:brand-copywriting",
          "target": "domains/brand/knowledge/tone-of-voice.md",
          "kind": "source_usage",
          "labels": ["tone guidance"],
          "contexts": ["primary source of brand voice constraints"],
          "occurrenceCount": 1
        }
      ]
    }
  },
  "sourceHashes": {
    "domains/brand/knowledge/tone-of-voice.md": "sha256:a1b2c3..."
  }
}
```

**Commit this file.** It provides the baseline hashes for [staleness detection](/staleness) and the graph data for the dashboard. Without it, `author stale` has nothing to compare against.

## `.agentpack/install.json`

Enabled-target state. Tracks which skill packages or exports are enabled for which agent runtimes.

**Produced by:** `agentpack materialize`, `agentpack skills enable`, `agentpack skills disable`

**Structure:**

```json
{
  "version": 2,
  "enabled_targets": {
    "@acme/brand-copywriting": ["agents", "claude"],
    "@acme/editorial-principles:style-guide": ["claude"]
  }
}
```

Each key is a direct installed package name or export id. The value is the list of runtimes it is selected for. When `materialize`, `skills enable`, or `skills disable` runs, agentpack reconciles the desired materializations from this state against the installed workspace graph.

**Do not commit.** This is runtime state specific to the consumer repo. Each repo has its own enabled set.

## `.agentpack/materialization-state.json`

Tracks which runtime files agentpack emitted and which adapter owns each one. Used for cleanup when skills are disabled or when the materialization target changes.

**Produced by:** `agentpack materialize`, `agentpack skills enable`, `agentpack skills disable`, `agentpack author dev`

**Structure:**

```json
{
  "version": 1,
  "generated_at": "2026-03-16T10:00:00.000Z",
  "adapters": {
    "claude": [
      {
        "packageName": "@acme/brand-copywriting",
        "skillName": "brand-copywriting",
        "runtimeName": "brand-copywriting",
        "sourceSkillPath": "node_modules/@acme/brand-copywriting",
        "sourceSkillFile": "node_modules/@acme/brand-copywriting/SKILL.md",
        "target": ".claude/skills/brand-copywriting",
        "mode": "symlink"
      }
    ],
    "agents": [
      {
        "packageName": "@acme/brand-copywriting",
        "skillName": "brand-copywriting",
        "runtimeName": "brand-copywriting",
        "sourceSkillPath": "node_modules/@acme/brand-copywriting",
        "sourceSkillFile": "node_modules/@acme/brand-copywriting/SKILL.md",
        "target": ".agents/skills/brand-copywriting",
        "mode": "symlink"
      }
    ]
  }
}
```

**Do not commit.** Runtime adapter state that is repo-specific and regenerated by `materialize`, `skills enable`, and `skills disable`.

## `.agentpack/dev-session.json`

Tracks the active `author dev` session: the process PID, the root skill, and all linked skill paths. Used for crash recovery and cleanup.

**Produced by:** `agentpack author dev`

**Structure:**

```json
{
  "version": 1,
  "session_id": "dev-2026-03-16T10-00-00-000Z",
  "status": "active",
  "pid": 12345,
  "repo_root": "/path/to/repo",
  "target": "path/to/skill",
  "root_skill": {
    "name": "brand-copywriting",
    "package_name": "@acme/brand-copywriting",
    "path": "path/to/skill"
  },
  "linked_skills": [
    {
      "name": "brand-copywriting",
      "package_name": "@acme/brand-copywriting",
      "path": "path/to/skill"
    }
  ],
  "links": [
    ".claude/skills/brand-copywriting",
    ".agents/skills/brand-copywriting"
  ],
  "started_at": "2026-03-16T10:00:00.000Z",
  "updated_at": "2026-03-16T10:00:00.000Z"
}
```

**Do not commit.** Ephemeral session state. If a dev session crashes, run `agentpack author dev cleanup` to remove stale links.
