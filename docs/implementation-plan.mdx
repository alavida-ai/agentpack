## Purpose

Translate the replacement architecture into a concrete implementation sequence.

The implementation now follows a layered path:

1. extract shared skill-domain modules first
2. route command workflows through application entrypoints
3. move file-backed state and runtime side effects behind infrastructure modules
4. preserve behavior through fixture-driven integration tests and focused domain tests

## Outcome

Deliver the first real `agentpack` slice with enough behavior to prove the architecture in production shape without overbuilding.

Success means:

- a workbench's local skills can imply external dependency installation
- one skill package plus transitive dependencies can be installed and materialized
- one plugin package can expose a validated bundle closure for vendored standalone skills
- stale skills can be detected from `sources`
- the repo environment can be inspected clearly
- generated metadata stays thin and legible

## Repo Strategy

The `agentpack` repo should contain:

- the CLI implementation
- unit tests for pure logic
- fixture-driven integration tests
- a compact test monorepo fixture

Real Alavida repos should be used for live validation only.

## Workstreams

### 1. Freeze Inputs

Lock before coding:

- architecture
- schema names
- first command slice
- repo-only install scope

### 2. Implement Core Models

Build:

- `SKILL.md` parser
- package metadata reader
- source hash computation
- skill identity resolver
- dependency graph and affected-state logic

Deliverables:

- stable in-memory models
- unit tests for parsing and hashing

### 3. Build Authoring Metadata

Build:

- catalog generation
- build-state generation
- stale comparison logic

Deliverables:

- `.agentpack/catalog.json`
- `.agentpack/build-state.json`
- integration test for stale detection after source change

### 4. Build Install And Materialization

Build:

- install root selection
- npm-backed install/update/uninstall orchestration
- materialization into `.claude/skills/` and `.agents/skills/`
- runtime reconciliation into `.agentpack/install.json`
- shared runtime watch/materialization helpers for both skills and plugin dev/build paths

Deliverables:

- install pipeline
- reconcile pipeline
- environment inspection support

### 5. Implement Command Surface

Commands:

1. `agentpack skills inspect`
2. `agentpack skills stale`
3. `agentpack skills install`
4. `agentpack skills env`
5. `agentpack plugin inspect`
6. `agentpack plugin validate`

Rule:

- read commands do not mutate state
- install is the only mutating command in v1

### 6. Validate On Fixtures

Maintain one compact fixture monorepo containing:

- `knowledge/`
- `skills/`
- `methodology/`
- `consumer/`
- `plugin/`

Use it to prove:

- catalog generation
- stale detection
- one direct install plus one dependency
- one plugin local skill closure plus one bundled transitive dependency
- materialization
- uninstall reconciliation

### 7. Validate On A Real Repo

After fixture coverage is stable:

- validate in the knowledge-base repo
- validate on one real workbench such as `website-dev`

## Explicitly Out Of Scope

- auto-rewriting stale skills
- publish orchestration
- marketplace orchestration
- dashboards
- migration tooling for the full old CLI
- global installs
