---
title: "Skills Commands"
description: "CLI reference for agentpack skills subcommands — runtime activation of installed skill packages."
---

```bash
agentpack skills <command> [options]
```

The `skills` command group manages installed-package inventory and runtime state inspection. The primary consumer materialization entrypoint is the top-level `agentpack materialize` command.

Package installation and removal are handled by npm directly. See [Publishing & Consuming Skills](/publishing) for the full workflow.

## Global flags

These flags work on every `agentpack` command, including all `skills` subcommands:

| Flag | Description |
|---|---|
| `--json` | Output as JSON. Useful for CI and scripting. |
| `-q, --quiet` | Suppress non-essential output. |
| `-v, --verbose` | Show detailed output. |
| `--workbench <path>` | Override workbench context (name or path). |
| `-V, --version` | Show version number. |

## Exit codes

All commands use consistent exit codes:

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | General error |
| `2` | Usage or validation error |
| `3` | Network error |
| `4` | Not found |

## `skills list`

List installed skill packages and their enabled runtimes.

```bash
agentpack skills list
```

No arguments. Scans `node_modules` for installed skill packages and shows each package with its exports and which runtimes they are enabled for.

```bash
agentpack skills list
```

---

## `materialize`

Materialize installed skill package dependencies for the current workspace package.

```bash
agentpack materialize
```

No arguments. Reads the nearest workspace `package.json`, selects installed dependencies that publish agentpack runtime skills, materializes them and their transitive skill requirements into `.claude/skills/` and `.agents/skills/`, and records the direct-package selection state in `.agentpack/install.json`.

```bash
agentpack materialize
agentpack materialize --json
```

<Note>
  Run this from the workspace package that owns the dependencies you want materialized. In nested workspaces, agentpack walks up to the nearest ancestor `package.json`.
</Note>

## `skills enable`

Materialize one installed skill package or export into runtime directories.

```bash
agentpack skills enable <target>
```

| Argument | Description |
|---|---|
| `target` | Required. Installed package name, export id, or installed skill path. |

| Option | Description |
|---|---|
| `-r, --runtime <runtime>` | Enable only for the selected runtime. Can be specified multiple times. Supported: `agents`, `claude`. |

Compatibility command for targeted activation. Resolves the target from `node_modules`, materializes the skill and its transitive dependencies into `.claude/skills/` and `.agents/skills/`, and records the enabled-target state in `.agentpack/install.json`.

```bash
# Enable for all runtimes
agentpack skills enable @acme/brand-copywriting

# Enable only for Claude
agentpack skills enable @acme/brand-copywriting -r claude
```

<Note>
  If your agent session was already running, start a fresh session to pick up newly enabled skills.
</Note>

---

## `skills disable`

Remove one installed skill package or export from runtime directories.

```bash
agentpack skills disable <target>
```

| Argument | Description |
|---|---|
| `target` | Required. Installed package name, export id, or installed skill path. |

| Option | Description |
|---|---|
| `-r, --runtime <runtime>` | Disable only for the selected runtime. Can be specified multiple times. Supported: `agents`, `claude`. |

Removes the target from the enabled set, reconciles runtime materializations, and updates `.agentpack/install.json`.

```bash
# Disable for all runtimes
agentpack skills disable @acme/brand-copywriting

# Disable only for agents runtime
agentpack skills disable @acme/brand-copywriting -r agents
```

## `skills status`

Show runtime health: installed vs enabled counts, selection issues, drift, and orphaned materializations.

```bash
agentpack skills status
```

No arguments. Reports installed package and export counts, enabled counts, selection issues (enabled targets no longer installed), runtime drift, and orphaned materializations.

```bash
agentpack skills status
```
