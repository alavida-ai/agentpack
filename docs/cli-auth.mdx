---
title: "Auth Commands"
description: "CLI reference for agentpack auth subcommands."
---

```bash
agentpack auth <command> [options]
```

The `auth` command group configures and inspects package registry authentication for private skill packages.

**Defaults:**
- Scope: `@alavida-ai`
- Registry: `https://npm.pkg.github.com`
- Token: GitHub personal access token with package read access

---

## `auth login`

Configure GitHub Packages authentication for this machine.

```bash
agentpack auth login [options]
```

| Option | Description |
|---|---|
| `--scope <scope>` | Override the package scope to configure. Default: `@alavida-ai`. |
| `--registry <url>` | Override the package registry URL. Default: `https://npm.pkg.github.com`. |
| `--verify-package <packageName>` | Override the package used for live verification. |

Prompts for a GitHub personal access token, writes the scope and registry configuration to your user `.npmrc`, and verifies the credential works against the configured registry.

```bash
# Default setup
agentpack auth login

# Custom scope and registry
agentpack auth login --scope @my-org --registry https://npm.pkg.github.com
```

---

## `auth status`

Show authentication status for the configured package registry.

```bash
agentpack auth status [options]
```

| Option | Description |
|---|---|
| `--verify` | Check whether the stored credential works against the configured registry. |

Without `--verify`, shows the local configuration state. With `--verify`, performs a live check against the registry.

```bash
# Local config check
agentpack auth status

# Live verification
agentpack auth status --verify
```

---

## `auth logout`

Remove configured package registry authentication.

```bash
agentpack auth logout
```

No options. Removes the stored credentials and npm registry keys from your `.npmrc`.

```bash
agentpack auth logout
```
