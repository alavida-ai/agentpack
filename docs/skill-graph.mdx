---
title: "The Skill Graph"
description: "How source bindings, skill imports, and contextual usage references form the compiled graph."
---

The skill graph is built from:

- source bindings in the `agentpack` block
- skill imports in the `agentpack` block
- explicit body references with `{context="..."}`

## Node types

- source nodes
- skill nodes

## Edge types

- source usage edges
- skill usage edges

## Example

```markdown
---
name: copywriting
description: Brand-consistent copywriting.
---

```agentpack
import research from skill "@alavida/research"
source tone = "domains/brand/knowledge/tone-of-voice.md"
```

Use [research](skill:research){context="required verification workflow before factual claims"}.
Apply [tone guidance](source:tone){context="primary source of brand voice constraints"}.
```

This yields:

- one skill edge from `copywriting` to `@alavida/research`
- one source edge from `copywriting` to `tone-of-voice.md`

## Inspecting the graph

Use:

```bash
agentpack skills inspect <target>
agentpack skills build <target>
agentpack skills dev <target>
```

The dashboard renders the aggregated graph, while compiled state keeps the exact occurrences and contexts.
