## Purpose

Describe what the replacement slice actually does today, separate from the broader architecture intent.

## Implemented Commands

### `agentpack plugin inspect`

Implemented and tested. Computes direct and transitive bundle closure for a plugin package from local skill `requires`.

### `agentpack plugin validate`

Implemented and tested. Verifies that plugin-local skill dependencies can be vendored successfully.

### `agentpack plugin build`

Implemented and tested. Produces a self-contained plugin artifact with vendored standalone skills.

### `agentpack plugin dev`

Implemented and tested. Builds the plugin artifact initially and rebuilds it on source changes.

### `agentpack skills inspect`

Implemented and tested. Supports packaged skill directory path, `SKILL.md` path, and package-name lookup within the repo.

### `agentpack skills validate`

Implemented and tested. Supports validating one packaged skill or all authored packaged skills in the repo. Valid packages return explicit npm release guidance.

### `agentpack skills dev`

Implemented and tested. Links one packaged skill into `.claude/skills/` and `.agents/skills/` for local runtime testing.

### `agentpack skills unlink`

Implemented and tested. Removes one dev-linked skill from local discovery paths.

### `agentpack skills outdated`

Implemented and tested. Supports repo-local update visibility for installed packaged skills with text and JSON output. Classifies updates as `patch`, `minor`, or `major`.

### `agentpack skills registry`

Implemented and tested. Supports repo-local `.npmrc` inspection for `@alavida` with text and JSON output.

### `agentpack skills status`

Implemented and tested. Provides environment health overview for a consumer repo with text and JSON output.

### `agentpack skills missing`

Implemented and tested. Identifies installed skills whose declared `requires` are not currently installed. Returns consumer-safe remediation commands.

### `agentpack skills dependencies`

Implemented and tested. Shows direct skill dependencies, reverse dependents, and lifecycle status (`current`, `stale`, `affected`).

### `agentpack skills stale`

Implemented and tested. Supports list and detail mode. Also includes deterministic skills catalog and build-state generation.

### `agentpack skills install`

Implemented and tested. Supports explicit packaged skill targets and workbench-implied roots from local workbench skill `requires`.

### `agentpack skills env`

Implemented and tested. Reports installed package names, direct vs transitive, version, lifecycle status, and materialized targets.

### `agentpack skills uninstall`

Implemented and tested. Runs `npm uninstall`, rebuilds runtime state, removes dead materializations.

## Current Scope Boundary

This slice is real and working, but still intentionally focused on repo-local authoring, installation, and plugin artifact workflows.

**Implemented:**

- plugin bundle graph inspection
- plugin bundle contract validation
- plugin artifact build and dev watch
- package validation for release-readiness
- local packaged-skill dev linking and unlinking
- registry-backed and repo-local available update visibility for installed skills
- repo-local private registry configuration visibility
- environment-level health overview for consumer repos
- dependency visibility for authored and installed skill graphs
- package-backed install model
- stale detection from recorded source hashes
- deterministic authoring metadata generation for catalog and build-state
- runtime materialization
- runtime state reconciliation
- reinstall stability for the same direct target
- failure-mode handling for missing install context and empty workbench roots
- multiple direct roots across separate installs
- shared-dependency preservation when one direct root is removed
- workbench bootstrap from multiple local skills with root union and deduplication

**Metadata policy:**

- commit `.agentpack/build-state.json`
- commit `.agentpack/catalog.json`
- do not commit `.agentpack/install.json`

**Not yet implemented:**

- explicit `agentpack plugin bundle` and `agentpack plugin pack` commands
- npm-backed publish orchestration for bundled plugin packages
- richer dependency-graph inspection
- removal of legacy primitives-oriented command groups

## Live Validation

Live validation is no longer limited to the original install/stale slice. The implemented CLI now also supports local packaged-skill dev links and bundled plugin build/dev workflows.

## Maintenance Rule

New behavior in the replacement path should be added only with:

1. a checked-in fixture or fixture update
2. a failing integration test
3. the smallest implementation needed to pass
4. a docs update if command behavior or state shape changed
