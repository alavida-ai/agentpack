title: "Quickstart"
description: "Install agentpack and see a compiler-mode skill graph in under 5 minutes."
---

Go from zero to a running skill with the visual dashboard open in under 5 minutes. You need Node.js 20+, npm, and a git repository.

<Steps>
  <Step title="Install agentpack">
    ```bash
    npm install -g @alavida/agentpack
    agentpack --version
    ```
  </Step>

  <Step title="Create a skill package">
    A minimal compiler-mode skill package needs `SKILL.md` and `package.json` in the same directory.

    **`skills/hello-world/SKILL.md`**
    ````markdown
    ---
    name: hello-world
    description: A minimal skill to verify agentpack is working.
    ---

    ```agentpack
    ```

    Respond with a friendly greeting when the user says hello.
    ````

    **`skills/hello-world/package.json`**
    ```json
    {
      "name": "@acme/hello-world",
      "version": "0.1.0"
    }
    ```
  </Step>

  <Step title="Run dev mode">
    From your repo root, point `skills dev` at the skill package:

    ```bash
    agentpack skills dev skills/hello-world
    ```

    agentpack compiles the skill, links it into `.claude/skills/` and `.agents/skills/`, and starts the workbench.
  </Step>

  <Step title="See what the agent sees">
    The skill is now materialized. Check what the agent will discover at runtime:

    ```bash
    ls .claude/skills/
    cat .claude/skills/hello-world.md
    ```

    The materialized file is the final output agents read at session start.
  </Step>

  <Step title="Explore the dashboard">
    The dashboard renders your compiled skill graph. Click the node to open the inspector panel showing source bindings, skill imports, and lifecycle status. Use the action buttons to validate or refresh.
  </Step>
</Steps>

<Tip>
  If you prefer a CLI-only workflow, pass `--no-dashboard`:

  ```bash
  agentpack skills dev --no-dashboard skills/hello-world
  ```
</Tip>

## What just happened?

- `SKILL.md` stayed the authored artifact.
- The `agentpack` block established the compiler contract, even when empty.
- `skills dev` compiled the package into `.agentpack/compiled.json`.
- Runtime outputs were recorded in `.agentpack/materialization-state.json`.
- The dashboard visualized the compiled graph instead of inferring relationships from prose.

## Troubleshooting

<AccordionGroup>
  <Accordion title="agentpack: command not found">
    Make sure you installed globally: `npm install -g @alavida/agentpack`. Verify with `agentpack --version`. If using `npx`, run `npx @alavida/agentpack skills dev skills/hello-world`.
  </Accordion>
  <Accordion title="SKILL.md missing required agentpack declarations block">
    Every SKILL.md needs a fenced code block with the `agentpack` language tag. Even if you have no sources or imports, include an empty block.
  </Accordion>
  <Accordion title="Dashboard doesn't open">
    Pass `--no-dashboard` if your environment doesn't support opening a browser. The CLI workflow works without it.
  </Accordion>
</AccordionGroup>
