## What you'll learn

You'll add a dependency from one exported skill module to another, see how agentpack keeps package dependencies aligned, and visualize the graph in both the dashboard and CLI.

**Scenario:** Your `copywriting` skill depends on a `research` skill exported from another package. You want the package contract and the skill graph to agree.

## Prerequisites

- Completed [Create Your First Skill](/first-skill)
- A `research` skill package in your repo (we'll create one below)

<Steps>

<Step title="Create the research skill">

First, create the package that exports the `research` skill.

```bash
mkdir -p domains/brand/knowledge
mkdir -p domains/brand/skills/research/skills/research
```

```markdown domains/brand/knowledge/research-principles.md
# Research Principles

- Always cite the primary source, not a summary
- Verify claims against at least two independent sources
- Prefer recent data (< 2 years old) for market claims
```

```markdown domains/brand/skills/research/skills/research/SKILL.md
---
name: research
description: Product and market research using company research standards
metadata:
  sources:
    - domains/brand/knowledge/research-principles.md
requires: []
---

## Instructions

When researching for content, follow these principles:

1. Start with primary sources. Never cite a summary without checking the original.
2. Cross-reference claims against at least two independent sources.
3. Flag any data older than two years as potentially outdated.
```

```json domains/brand/skills/research/package.json
{
  "name": "@alavida/research",
  "version": "1.0.0",
  "description": "Product and market research using company research standards",
  "files": ["skills"],
  "agentpack": {
    "skills": {
      "research": {
        "path": "skills/research/SKILL.md"
      }
    }
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/your-org/knowledge-base.git"
  },
  "publishConfig": {
    "registry": "https://npm.pkg.github.com"
  }
}
```

</Step>

<Step title="Add the requires declaration">

Open your copywriting skill's `SKILL.md` and add the research skill to `requires`:

```markdown domains/brand/skills/copywriting/skills/copywriting/SKILL.md
---
name: copywriting
description: Brand-consistent copywriting using company tone and voice principles
metadata:
  sources:
    - domains/brand/knowledge/tone-of-voice.md
requires:
  - "@alavida/research:research"
---

## Instructions

When writing copy for the brand, follow these guidelines:

1. Use the research skill to verify any product claims before writing.
2. Read and internalize the tone-of-voice principles before drafting.
3. Use active voice throughout. Address the reader as "you."
4. Avoid jargon unless you explain it in the same sentence.
```

The `requires` array takes canonical skill ids, not bare package names.

Format:

- `@scope/package:skill-name`

The package part tells npm what must be installed. The `:skill-name` part tells agentpack which exported module inside that package is required.

</Step>

<Step title="Watch agentpack sync the dependency">

Run `agentpack skills dev` on the copywriting package:

```bash
agentpack skills dev domains/brand/skills/copywriting
```

```
Linked Skill: copywriting
Path: domains/brand/skills/copywriting/skills/copywriting
Synced Added: 1
Synced Removed: 0
Linked Skills: 2
Linked: .claude/skills/copywriting
Linked: .claude/skills/research
Workbench URL: http://localhost:3000
```

Notice `Synced Added: 1`. agentpack detected the cross-package `requires` entry and automatically added `@alavida/research` to `package.json.dependencies`.

Check your `package.json` -- it now has a `dependencies` field:

```json domains/brand/skills/copywriting/package.json
{
  "name": "@alavida/copywriting",
  "version": "0.1.0",
  "description": "Brand-consistent copywriting using company tone and voice principles",
  "files": ["skills"],
  "agentpack": {
    "skills": {
      "copywriting": {
        "path": "skills/copywriting/SKILL.md"
      }
    }
  },
  "dependencies": {
    "@alavida/research": "*"
  }
}
```

<Note>
This is the **go mod tidy** pattern. `SKILL.md.requires` is the authored dependency truth. For cross-package references, `package.json.dependencies` is the managed package-level mirror. Add a cross-package requires entry and the package dependency appears. Remove it and the package dependency disappears. Same-package module references do not create package dependencies.
</Note>

</Step>

<Step title="View the dependency graph in the dashboard">

Open the dashboard URL from the dev output. You'll see two types of edges:

- **Source provenance edges** connecting the copywriting skill to `tone-of-voice.md`
- **Dependency edges** connecting the copywriting skill to the exported `@alavida/research:research` module

If both skills live in the same package, the dependency is an internal module edge. If they live in different packages, the edge crosses a package boundary and install pulls the external package.

</Step>

<Step title="Inspect dependencies via CLI">

Use `agentpack skills dependencies` to see the graph from the terminal:

```bash
agentpack skills dependencies domains/brand/skills/copywriting
```

```
Skill: @alavida/copywriting:copywriting
Graph: authored
Version: 0.1.0
Path: domains/brand/skills/copywriting/skills/copywriting/SKILL.md

Direct Dependencies:
- @alavida/research:research

Reverse Dependencies:
- none
```

Now check the research skill:

```bash
agentpack skills dependencies domains/brand/skills/research
```

```
Skill: @alavida/research:research
Graph: authored
Version: 1.0.0
Path: domains/brand/skills/research/skills/research/SKILL.md

Direct Dependencies:
- none

Reverse Dependencies:
- @alavida/copywriting:copywriting
```

The reverse dependency shows that `research` knows it is depended upon by `copywriting`.

</Step>

</Steps>

## Direct vs transitive dependencies

Dependencies form a graph. Consider this chain:

| Skill | Requires |
|---|---|
| `@alavida/copywriting:copywriting` | `@alavida/research:research` |
| `@alavida/research:research` | `@alavida/methodology-gary-provost:gary-provost` |
| `@alavida/methodology-gary-provost:gary-provost` | (nothing) |

From the perspective of `copywriting`:

- `@alavida/research:research` is a **direct** dependency
- `@alavida/methodology-gary-provost:gary-provost` is a **transitive** dependency

When a consumer runs `agentpack skills install @alavida/copywriting`, npm resolves the full package graph. agentpack then materializes every exported skill in the closure into `.claude/skills/` so agents can discover them all.

<Warning>
**Circular dependencies are not allowed.** If skill A requires skill B and skill B requires skill A, validation will fail. Design your skills as a directed acyclic graph (DAG) -- each skill should have a clear "upstream" direction.
</Warning>

## How the sync works under the hood

Every agentpack command that touches dependencies (`dev`, `validate`, `plugin build`) performs a total bidirectional sync:

```
1. Read SKILL.md.requires         → ["@alavida/research:research"]
2. Read package.json.dependencies → {}
3. Extract cross-package names    → ["@alavida/research"]
4. Add any missing package deps
5. Remove managed package deps no longer referenced
6. Write package.json
```

The result: package dependencies stay aligned with the cross-package closure declared in `requires`, while same-package module references stay internal to the package.
