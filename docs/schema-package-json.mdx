Every packaged skill module ships inside an npm package. The `package.json` provides the package identity, version, and distribution metadata that npm needs to publish and install it.

## Skill package fields

| Field | Required | Description |
|---|---|---|
| `name` | Yes | Scoped package name (e.g., `@acme/brand-copywriting`). |
| `version` | Yes | Semver version (e.g., `1.2.0`). |
| `description` | Recommended | Package-level summary. |
| `files` | Yes | Which files to include in the published package. Must include `SKILL.md` and the discovery root directory if applicable. |
| `dependencies` | Managed | Required for cross-package skill references. Managed from exported skill imports. |
| `publishConfig` | For managed scopes | Registry configuration for GitHub Packages. Required for `@alavida`/`@alavida-ai` scoped packages. |
| `repository` | For managed scopes | Repository URL for the source package. Required for `@alavida`/`@alavida-ai` scoped packages. |

## Export discovery

agentpack discovers skill exports from the package filesystem, not from an explicit export map:

1. **Root `SKILL.md`** — a `SKILL.md` at the package root is the **primary export**. Its runtime name defaults to the package's unscoped name (e.g., `@acme/brand-copywriting` → `brand-copywriting`).
2. **Named exports via `skills/`** — if the package contains `skills/**/SKILL.md`, agentpack walks that subtree and exports each nested skill directory as a named export.

### Single-skill package (most common)

```json package.json
{
  "name": "@acme/brand-copywriting",
  "version": "1.2.0",
  "description": "Brand copywriting package.",
  "files": ["SKILL.md"],
  "repository": {
    "type": "git",
    "url": "git+https://github.com/acme-corp/knowledge-base.git"
  },
  "publishConfig": {
    "registry": "https://npm.pkg.github.com"
  }
}
```

The root `SKILL.md` is the only export.

### Multi-skill package

```json package.json
{
  "name": "@acme/brand-suite",
  "version": "1.2.0",
  "description": "Brand skill suite.",
  "files": ["SKILL.md", "skills/"],
  "repository": {
    "type": "git",
    "url": "git+https://github.com/acme-corp/knowledge-base.git"
  },
  "publishConfig": {
    "registry": "https://npm.pkg.github.com"
  }
}
```

This discovers:
- `SKILL.md` → primary export (`@acme/brand-suite`)
- `skills/value-copywriting/SKILL.md` → named export (`@acme/brand-suite:value-copywriting`)
- `skills/editorial-principles/SKILL.md` → named export (`@acme/brand-suite:editorial-principles`)

## What package.json owns

package.json is the source of truth for:

- **Package version** — the semver version number. agentpack reads it but never writes it. You bump it yourself or with `npm version`.
- **Publish configuration** — the registry URL and any access settings.
- **Repository URL** — identifies where the source package lives.
- **Export convention** — the root `SKILL.md` is the primary export and `skills/**/SKILL.md` provides named exports.
- **Files list** — controls what gets published. Keep it tight: just the authored exports you need and the built `dist/` output you intend consumers to install.

## What package.json does NOT own

| Concern | Owned by |
|---|---|
| Skill identity and content | `SKILL.md` frontmatter + body |
| Source provenance | `SKILL.md` `source` bindings |
| Skill dependencies | `SKILL.md` `import` statements (compiled into cross-package `package.json.dependencies`) |
| Export discovery paths | Filesystem layout (root `SKILL.md` + optional `skills/**/SKILL.md`) |

<Warning>
  Do not manually edit managed cross-package skill dependencies in `dependencies`. agentpack derives them from exported skill imports. Manual edits will be overwritten on the next `author dev` run.
</Warning>

## The dependency sync model

agentpack syncs dependencies the way `go mod tidy` syncs `go.mod`:

1. Read cross-package skill imports from each exported `SKILL.md`
2. Read `dependencies` from package.json
3. Add cross-package requirements that are missing from `dependencies`
4. Remove managed-scope dependencies (`@alavida`, `@alavida-ai`) that are no longer required by those imports — non-managed dependencies are never touched
5. Write package.json

This sync runs automatically inside `agentpack author dev`. `agentpack publish validate` checks that dependencies are present but does not write them. You never run sync separately.

## Validation checks

`agentpack publish validate` checks these package.json conditions:

- `name` and `version` are present
- Each discovered skill export (`SKILL.md` files) compiles successfully
- `files` includes the exported skill paths (if the `files` field exists)
- Every cross-package skill import from exported skills appears in `dependencies`
- Every declared source file exists on disk
- `@alavida`/`@alavida-ai` scoped packages include a `repository` field
- `@alavida`/`@alavida-ai` scoped packages set `publishConfig.registry` to `https://npm.pkg.github.com`
