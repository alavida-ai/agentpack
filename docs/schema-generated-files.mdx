---
title: "Generated Files Reference"
description: "Reference for the three generated files agentpack produces: build-state.json, catalog.json, and install.json."
---

agentpack generates three files in the `.agentpack/` directory. Two should be committed to your repository. One should not.

| File | Commit? | Purpose |
|---|---|---|
| `.agentpack/build-state.json` | Yes | Source hashes for stale detection |
| `.agentpack/catalog.json` | Yes | Index of all skills in the repo |
| `.agentpack/install.json` | **No** | Runtime install state (environment-specific) |

---

## build-state.json

**Purpose:** Records source file hashes at validation time so agentpack can detect when sources change and skills become stale.

**Generated by:** `agentpack skills validate`

**Commit this file.** It is generated metadata, not ephemeral cache. Diffs are useful because they show provenance changes over time. Committing it enables stale detection across CI, teammates, and fresh clones.

### Example

```json .agentpack/build-state.json
{
  "version": 1,
  "skills": {
    "@acme/brand-copywriting": {
      "package_version": "1.2.0",
      "skill_path": "domains/brand/skills/copywriting",
      "skill_file": "domains/brand/skills/copywriting/SKILL.md",
      "sources": {
        "domains/brand/knowledge/tone-of-voice.md": {
          "hash": "sha256:8c4d2f1a..."
        },
        "domains/brand/knowledge/selling-points.md": {
          "hash": "sha256:4aa19bc3..."
        }
      },
      "requires": [
        "@acme/editorial-principles"
      ]
    }
  }
}
```

### Fields

| Field | Description |
|---|---|
| `version` | Schema version (currently `1`). |
| `skills.<package>` | One entry per packaged skill, keyed by npm package name. |
| `package_version` | Version at last validation. |
| `skill_path` | Repo-relative directory path. |
| `skill_file` | Repo-relative SKILL.md path. |
| `sources.<path>.hash` | SHA-256 content hash of each source file at validation time. |
| `requires` | Direct dependency package names at validation time. |

<Tip>
  `agentpack skills stale` compares current source file hashes against the recorded hashes in this file. If a source changed since the last validation, the skill is marked stale.
</Tip>

## catalog.json

**Purpose:** A generated index of all authored skills in the repository. Enables discovery and inspection without parsing every SKILL.md individually.

**Generated by:** `agentpack skills validate` (when run without a target, validates and catalogs all skills)

**Commit this file.** It is the generated inventory of authored skills. It should stay deterministic and reviewable in diffs.

### Example

```json .agentpack/catalog.json
{
  "version": 1,
  "skills": {
    "@acme/brand-copywriting": {
      "name": "brand-copywriting",
      "description": "Write brand-consistent copy using company tone and voice guidelines.",
      "path": "domains/brand/skills/copywriting",
      "skill_file": "domains/brand/skills/copywriting/SKILL.md",
      "package_name": "@acme/brand-copywriting",
      "package_version": "1.2.0",
      "sources": [
        "domains/brand/knowledge/tone-of-voice.md",
        "domains/brand/knowledge/selling-points.md"
      ],
      "requires": [
        "@acme/editorial-principles"
      ]
    }
  }
}
```

### Fields

| Field | Description |
|---|---|
| `version` | Schema version (currently `1`). |
| `skills.<package>.name` | Logical skill name from SKILL.md frontmatter. |
| `skills.<package>.description` | Skill description. |
| `skills.<package>.path` | Repo-relative skill directory. |
| `skills.<package>.skill_file` | Repo-relative SKILL.md path. |
| `skills.<package>.package_name` | npm package name. |
| `skills.<package>.package_version` | Current package version. |
| `skills.<package>.sources` | Array of declared source paths. |
| `skills.<package>.requires` | Array of direct dependency package names. |

## install.json

**Purpose:** Tracks runtime install state for skill packages in a consumer repo. Records which skills are installed, whether they are direct or transitive, and where they were materialized.

**Generated by:** `agentpack skills install` and `agentpack skills uninstall`

<Warning>
  **Do NOT commit this file.** It is environment-specific runtime state. Each developer's install paths and materialization modes may differ. Add it to `.gitignore`.
</Warning>

```gitignore .gitignore
.agentpack/install.json
```

### Example

```json .agentpack/install.json
{
  "version": 1,
  "installs": {
    "@acme/brand-copywriting": {
      "direct": true,
      "package_version": "1.2.0",
      "source_package_path": "node_modules/@acme/brand-copywriting",
      "materializations": [
        {
          "target": ".claude/skills/brand-copywriting",
          "mode": "symlink"
        },
        {
          "target": ".agents/skills/brand-copywriting",
          "mode": "symlink"
        }
      ]
    },
    "@acme/editorial-principles": {
      "direct": false,
      "package_version": "1.0.0",
      "source_package_path": "node_modules/@acme/editorial-principles",
      "materializations": [
        {
          "target": ".claude/skills/editorial-principles",
          "mode": "symlink"
        }
      ]
    }
  }
}
```

### Fields

| Field | Description |
|---|---|
| `version` | Schema version (currently `1`). |
| `installs.<package>.direct` | `true` if explicitly installed, `false` if pulled in as a transitive dependency. |
| `installs.<package>.package_version` | Installed version. |
| `installs.<package>.source_package_path` | Path to the resolved package in `node_modules/`. |
| `installs.<package>.materializations` | Array of locations where the skill was placed for agent discovery. |
| `materializations[].target` | Repo-relative path (e.g., `.claude/skills/brand-copywriting`). |
| `materializations[].mode` | How the skill was placed: `"symlink"` or `"copy"`. |
