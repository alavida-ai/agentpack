## What you'll build

You'll create a **copywriting skill** that teaches agents to write in your brand's tone. By the end, the skill will be validated, linked for local development, and visible in the agentpack dashboard.

**Scenario:** Your team has brand guidelines in markdown files. You want every agent in the repo to follow them consistently, without copy-pasting instructions into every prompt.

## Prerequisites

- Node.js 18+
- agentpack installed globally: `npm install -g @alavida/agentpack`
- A git repository to work in

## Directory layout

Here is what you'll create by the end of this tutorial:

```
your-repo/
├── domains/
│   └── brand/
│       ├── knowledge/
│       │   └── tone-of-voice.md        ← source knowledge (the truth)
│       └── skills/
│           └── copywriting/
│               ├── SKILL.md            ← compiled skill artifact
│               └── package.json        ← distribution metadata
└── .claude/
    └── skills/
        └── copywriting/                ← symlink (created by agentpack skills dev)
```

<Steps>

<Step title="Write the source knowledge">

Source knowledge files are the raw truth your skill is built from. These are the markdown documents your team already maintains -- brand guides, style docs, research principles.

Create the knowledge directory and write a tone-of-voice guide:

```bash
mkdir -p domains/brand/knowledge
```

```markdown tone-of-voice.md
# Tone of Voice

## Principles

- **Warm but direct.** We don't hedge. We speak to the reader like a trusted friend
  who happens to know the answer.
- **Confident, not arrogant.** State what we know. Acknowledge what we don't.
- **Active voice.** "We built this" not "This was built by us."

## Word choices

- Say "you" not "the user"
- Say "simple" not "easy" (nothing is easy when you're stuck)
- Say "works with" not "integrates with"

## What to avoid

- Jargon without explanation
- Exclamation marks in body copy
- Marketing superlatives ("best-in-class", "world-class", "cutting-edge")
```

<Tip>
Knowledge files are plain markdown. There is no special format -- just write what you'd put in a style guide or wiki page. The skill will reference these files as its sources.
</Tip>

</Step>

<Step title="Create the SKILL.md">

`SKILL.md` is the compiled skill artifact. Its frontmatter declares the skill's identity, its source provenance, and any dependencies. The body contains the instructions agents will follow.

```bash
mkdir -p domains/brand/skills/copywriting
```

```markdown SKILL.md
---
name: copywriting
description: Brand-consistent copywriting using company tone and voice principles
metadata:
  sources:
    - domains/brand/knowledge/tone-of-voice.md
requires: []
---

## Instructions

When writing copy for the brand, follow these guidelines:

1. Read and internalize the tone-of-voice principles before drafting.
2. Use active voice throughout. Address the reader as "you."
3. Avoid jargon unless you explain it in the same sentence.
4. Never use marketing superlatives.
5. Keep sentences varied in length. Mix short and long for rhythm.

## When to use this skill

- Landing page copy
- Product descriptions
- Email sequences
- Help center articles

## When NOT to use this skill

- Internal engineering docs (use a technical writing skill instead)
- Legal or compliance copy (requires legal review)
```

The `metadata.sources` array lists the knowledge files this skill was built from. agentpack uses these paths for provenance tracking and stale detection -- if someone updates `tone-of-voice.md`, agentpack knows this skill may need a refresh.

</Step>

<Step title="Create the package.json">

Every skill is an npm package. The `package.json` provides the identity and version that npm needs for distribution.

```json package.json
{
  "name": "@alavida/copywriting",
  "version": "0.1.0",
  "description": "Brand-consistent copywriting using company tone and voice principles",
  "files": ["SKILL.md"],
  "repository": {
    "type": "git",
    "url": "git+https://github.com/your-org/knowledge-base.git"
  },
  "publishConfig": {
    "registry": "https://npm.pkg.github.com"
  }
}
```

<Note>
The `files` array must include `SKILL.md`. This is what gets published when you `npm publish` -- the skill artifact, not the source knowledge.
</Note>

</Step>

<Step title="Validate the skill">

Run `agentpack skills validate` to check that the skill is structurally sound and ready for use.

```bash
agentpack skills validate domains/brand/skills/copywriting
```

Expected output:

```
Skill: @alavida/copywriting
Status: valid
Issues: 0
```

If validation fails, the output tells you exactly what to fix -- missing fields, unresolvable sources, or mismatched dependencies.

</Step>

<Step title="Start dev mode">

Run `agentpack skills dev` to link the skill into the agent discovery directories and launch the visual dashboard.

```bash
agentpack skills dev domains/brand/skills/copywriting
```

Expected output:

```
Linked Skill: copywriting
Path: domains/brand/skills/copywriting
Synced Added: 0
Synced Removed: 0
Linked Skills: 1
Linked: .claude/skills/copywriting
Workbench URL: http://localhost:3000
Note: if your current agent session was already running, start a fresh session
to pick up newly linked skills.
```

This does three things:
1. **Symlinks** the skill into `.claude/skills/copywriting` and `.agents/skills/copywriting` so agents can discover it
2. **Starts the dashboard** at `http://localhost:3000` with a visual skill graph
3. **Watches** your skill directory and source files for changes -- edits trigger an automatic reload

{/* Screenshot: dashboard showing a single "copywriting" skill node with a provenance edge connecting it to the tone-of-voice.md source file */}

<Tip>
If you only want the symlinks without the dashboard, pass `--no-dashboard`:

```bash
agentpack skills dev --no-dashboard domains/brand/skills/copywriting
```
</Tip>

The dev process keeps running. When you stop it (Ctrl+C), the symlinks are automatically removed.

</Step>

<Step title="Inspect the skill">

In a separate terminal, run `agentpack skills inspect` to see the full skill metadata:

```bash
agentpack skills inspect domains/brand/skills/copywriting
```

Expected output:

```
Skill: copywriting
Description: Brand-consistent copywriting using company tone and voice principles
Package: @alavida/copywriting
Version: 0.1.0
Path: domains/brand/skills/copywriting/SKILL.md

Sources:
- domains/brand/knowledge/tone-of-voice.md

Requires:
- none
```

This is the same information the dashboard displays, in a terminal-friendly format. Use `--json` for machine-readable output.

{/* Screenshot: terminal showing the agentpack skills inspect output for the copywriting skill */}

</Step>

</Steps>

## What just happened

You created a skill using the agentpack compiler model:

| Step | What it is |
|---|---|
| `tone-of-voice.md` | Source knowledge (the truth your team maintains) |
| `SKILL.md` | Compiled skill artifact (instructions + provenance metadata) |
| `package.json` | Distribution metadata (npm package identity) |
| `.claude/skills/copywriting` | Runtime materialization (how agents discover it) |

Your skill is now discoverable by any agent in this repo. Start a new Claude Code or Codex session, and the agent will see the copywriting skill and follow its instructions.
