## What you'll build

You'll create a **copywriting package** with one exported `copywriting` skill. By the end, it will be validated, compiled, linked for local development, and visible in the dashboard.

## Directory layout

```text
your-repo/
├── domains/
│   └── brand/
│       ├── knowledge/
│       │   └── tone-of-voice.md
│       └── skills/
│           └── copywriting/
│               ├── package.json
│               └── SKILL.md
└── .claude/
    └── skills/
```

<Steps>
  <Step title="Write the source knowledge">
    ```markdown
    # Tone of Voice

    - Warm but direct
    - Active voice always
    - Avoid unexplained jargon
    ```
  </Step>

  <Step title="Create the skill">
    ```markdown
    ---
    name: copywriting
    description: Brand-consistent copywriting using company tone guidance.
    ---

    ```agentpack
    source tone = "domains/brand/knowledge/tone-of-voice.md"
    ```

    Apply [tone guidance](source:tone){context="primary source of tone constraints for customer-facing copy"}.
    ```
  </Step>

  <Step title="Create the package metadata">
    ```json
    {
      "name": "@alavida/copywriting",
      "version": "0.1.0",
      "files": ["SKILL.md"],
      "repository": {
        "type": "git",
        "url": "git+https://github.com/your-org/knowledge-base.git"
      },
      "publishConfig": {
        "registry": "https://npm.pkg.github.com"
      }
    }
    ```
  </Step>

  <Step title="Validate and build">
    ```bash
    agentpack skills validate domains/brand/skills/copywriting
    agentpack skills build @alavida/copywriting
    ```
  </Step>

  <Step title="Run dev mode">
    ```bash
    agentpack skills dev domains/brand/skills/copywriting
    ```

    This compiles the skill, materializes runtime links, and starts the dashboard.
  </Step>
</Steps>

## What agentpack manages

- `.agentpack/compiled.json` for semantic state
- `.agentpack/materialization-state.json` for runtime ownership
- `.claude/skills/` and `.agents/skills/` as runtime outputs
