---
title: "Staleness Detection"
description: "How agentpack tracks whether compiled skills still match their bound sources."
---

Staleness detection compares the current content hash of each bound source file against the hash recorded in `.agentpack/compiled.json`.

## The model

- source file changed: the source node is `changed`
- skill bound to that source: the skill is `stale`
- skills importing that stale skill: they become `affected`

## Check for staleness

```bash
agentpack skills stale
agentpack skills stale @scope/package
```

If compiled state does not exist yet, build or validate first:

```bash
agentpack skills build <target>
```

## Fixing a stale skill

1. Update the source-backed `SKILL.md`
2. Re-run:

```bash
agentpack skills validate <target>
```

or

```bash
agentpack skills build <target>
```

3. Re-materialize if needed:

```bash
agentpack skills materialize
```

## Shared state

Commit `.agentpack/compiled.json` when you want staleness and semantic graph state reviewed or shared in CI.
