State Over Time and Required Artifact Files

What changes across the lifecycle, and which files must exist to keep the system legible

Lifecycle Over Time
1. Source Truth

Knowledge + Authored Skill

Humans update source files and the skill artifact that points at them.

Key files: knowledge docs, SKILL.md, package.json

2. Validate

Lifecycle Snapshot

The CLI checks rules and records the source-hash snapshot for the validated skill.

Key output: .agentpack/build-state.json

↓
3. Drift Happens

Sources Change

A knowledge file changes after validation. The skill artifact may now be behind reality.

Comparison: current source hashes vs recorded build-state

4. Stale Detection

Lifecycle Warning

skills stale reports the skill as stale. Reverse dependents may become affected.

Key owner: skill provenance + skill graph

↓
5. Install / Materialize

Runtime State

Consumer repos install packages and materialize agent-visible skill directories.

Key output: .agentpack/install.json

6. Plugin Bundle

Distribution Artifact

Plugins vendor packaged skills into a self-contained runtime artifact.

Key output: .agentpack/dist/plugins/.../

Minimal Required Artifact Files

SKILL.md

The authored skill artifact. Owns instructions, metadata.sources, and requires.

package.json

The distribution contract. Owns package identity, version, dependencies, and publish settings.

.agentpack/catalog.json

Generated authoring index for discovery and inspection across known packaged skills.

.agentpack/build-state.json

Generated provenance snapshot. Required for stale detection and explainable lifecycle state.

.agentpack/install.json

Environment-local runtime state for direct/transitive installs and materialization targets.

.claude-plugin/plugin.json

Plugin runtime contract. Separate from package versioning and skill lifecycle truth.

.claude-plugin/bundled-skills.json

Generated plugin provenance for which packaged skills were vendored into the bundle.

File Ownership Rules

Must stay explicit

  • SKILL.md owns authored dependency truth.
  • package.json owns distributable package truth.
  • build-state.json owns recorded source hashes.
  • install.json owns environment-local install/materialization state.
  • Plugin files own plugin runtime packaging only.

Must not blur together

  • Do not let plugin metadata become the source of skill dependency truth.
  • Do not let install state act like provenance state.
  • Do not let command handlers write state ad hoc.
  • Do not let build-state be owned by one command path only.
