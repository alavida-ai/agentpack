Graph Layout Direction

Your reference image uses left-to-right. But for a skill developer focused on one skill, which layout feels most natural?

A. Left-to-Right Tree

Like your reference image. Sources flow left → selected skill → dependencies → transitive deps. Natural reading direction, matches your existing mental model.

B. Radial / Center-Out

Selected skill at center, sources on one side, dependencies radiating outward. Emphasizes "this is my skill's world." Good for seeing the full picture at once, but can get cluttered with many nodes.

C. Top-Down DAG

Sources feed down into selected skill, which flows down to dependencies. Feels like a "waterfall" of data. Familiar from dependency diagrams, but horizontal space can get tight with many siblings.
