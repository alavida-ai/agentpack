## Purpose

Run the first-slice package-backed lifecycle against one real workbench in one real Alavida repo.

This is not the primary automated harness. Fixtures remain the primary safety net.

For a repo-local demo and manual validation path without leaving this repository, initialize submodules and use `sandbox/acme-demo/`.

## Target

- repo: knowledge-base monorepo
- workbench: `domains/value/workbenches/consumer/website-dev`

## Script

Run:

```bash
npm run validate:live -- --repo /path/to/knowledge-base/Alavida
```

For a more generic monorepo smoke run against your own repo:

```bash
npm run smoke:monorepo -- \
  --repo /path/to/your/monorepo \
  --inspect-target domains/value/workbenches/consumer/website-dev/skills/copywriting \
  --install-target /path/to/your/monorepo/workspace/active/architecture/intent-adoption/spike/packages/value-copywriting \
  --touch-source domains/value/knowledge/value-proposition.md
```

The script:

1. regenerates `.agentpack/catalog.json` and `.agentpack/build-state.json` in the target repo
2. inspects one real local workbench skill
3. infers real dependency roots from the workbench
4. installs and materializes the real package chain
5. checks `skills env`
6. changes one real source file temporarily and verifies `skills stale`
7. restores the source file
8. uninstalls and reconciles runtime state
9. cleans up runtime artifacts from the target repo

## Expected Scope

This script validates one minimal real package chain in `website-dev`:

- local workbench skill: `copywriting`
- direct external package: `@alavida/value-copywriting`
- transitive dependency: `@alavida/methodology-gary-provost`

It does not claim that the whole workbench has been migrated. It proves that the package-backed lifecycle works in a real repo and a real workbench context.
