## Purpose

Define the initial schema shapes for the replacement `agentpack`.

This covers:

- authored skill metadata in `SKILL.md`
- generated authoring metadata in `.agentpack/build-state.json`
- generated authoring catalog in `.agentpack/catalog.json`
- repo-local runtime state in `.agentpack/install.json`

## 1. `SKILL.md` Frontmatter

Required:

```yaml
---
name: value-copywriting
description: Write copy aligned with Alavida's value messaging and tone.
---
```

Optional agentpack fields:

```yaml
---
name: value-copywriting
description: Write copy aligned with Alavida's value messaging and tone.
metadata:
  sources:
    - domains/value/knowledge/selling-points.md
    - domains/value/knowledge/tone-of-voice.md
  status: deprecated
  replacement: "@alavida/value-research"
  message: Use the newer research-backed writing workflow.
requires:
  - "@alavida/methodology-gary-provost"
---
```

Rules:

- `name` is required
- `description` is required
- `metadata.sources` is an optional array of repo-relative paths
- `metadata.status` is optional and currently supports `deprecated` or `retired`
- `metadata.replacement` is an optional npm package name pointing at the successor skill
- `metadata.message` is an optional human-readable migration note
- `requires` is an optional array of npm package names
- `metadata.sources` order is preserved as authored
- `requires` stores direct dependencies only

## 2. `package.json` Fields agentpack Reads

agentpack depends on a minimal subset of package metadata:

```json
{
  "name": "@alavida/value-copywriting",
  "version": "1.2.0",
  "description": "Write copy aligned with Alavida's value messaging and tone.",
  "files": ["SKILL.md"],
  "dependencies": {
    "@alavida/methodology-gary-provost": "^1.0.0"
  }
}
```

Fields read by agentpack:

- `name`
- `version`
- `description`
- `dependencies`
- `files`

Mapping rule:

- `SKILL.md.requires` is the authored dependency source
- `package.json.dependencies` is the compiled install-time form

## 3. `.agentpack/build-state.json`

Purpose:

- record the last known build and provenance state of authored skill packages
- support stale detection
- support explainable inspection

Example:

```json
{
  "version": 1,
  "skills": {
    "@alavida/value-copywriting": {
      "package_version": "1.2.0",
      "skill_path": "domains/value/skills/copywriting",
      "skill_file": "domains/value/skills/copywriting/SKILL.md",
      "sources": {
        "domains/value/knowledge/selling-points.md": {
          "hash": "sha256:8c4d..."
        },
        "domains/value/knowledge/tone-of-voice.md": {
          "hash": "sha256:4aa1..."
        }
      },
      "requires": [
        "@alavida/methodology-gary-provost"
      ]
    }
  }
}
```

Rules:

- generated only
- one record per packaged skill or report
- missing record means no recorded build provenance yet
- stale detection compares current source hash to recorded source hash

Commit policy:

- commit this file in the authoring repo
- it is generated metadata, not ephemeral cache
- diffs are useful because they show provenance changes over time

## 4. `.agentpack/catalog.json`

Purpose:

- generated authoring-side discovery and summary index

Example:

```json
{
  "version": 1,
  "skills": {
    "@alavida/value-copywriting": {
      "name": "value-copywriting",
      "description": "Write copy aligned with Alavida's value messaging and tone.",
      "path": "domains/value/skills/copywriting",
      "skill_file": "domains/value/skills/copywriting/SKILL.md",
      "package_name": "@alavida/value-copywriting",
      "package_version": "1.2.0",
      "sources": [
        "domains/value/knowledge/selling-points.md",
        "domains/value/knowledge/tone-of-voice.md"
      ],
      "requires": [
        "@alavida/methodology-gary-provost"
      ]
    }
  }
}
```

Minimum catalog fields:

- logical skill name
- description
- repo-relative location
- package identity
- package version
- declared `sources`
- declared `requires`

Commit policy:

- commit this file in the authoring repo
- it is the generated inventory of authored skills
- it should stay deterministic and reviewable in diffs

## 5. `.agentpack/install.json`

Purpose:

- store repo-local runtime state for installed and materialized skills

Example:

```json
{
  "version": 1,
  "installs": {
    "@alavida/value-copywriting": {
      "direct": true,
      "package_version": "1.2.0",
      "source_package_path": "node_modules/@alavida/value-copywriting",
      "materializations": [
        {
          "target": ".claude/skills/value-copywriting",
          "mode": "symlink"
        },
        {
          "target": ".agents/skills/value-copywriting",
          "mode": "symlink"
        }
      ]
    },
    "@alavida/methodology-gary-provost": {
      "direct": false,
      "package_version": "1.0.0",
      "source_package_path": "node_modules/@alavida/methodology-gary-provost",
      "materializations": [
        {
          "target": ".claude/skills/gary-provost",
          "mode": "symlink"
        }
      ]
    }
  }
}
```

Rules:

- runtime-only state
- lockfile remains the source of full package graph truth
- `direct` distinguishes requested roots from transitive installs
- materializations must be explicit and inspectable

Commit policy:

- do not commit this file
- it is repo-local runtime state
- it is closer to environment state than authored truth
