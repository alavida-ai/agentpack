title: "Authentication"
description: "Set up npm registry authentication for installing and publishing private skill packages."
---

agentpack distributes skills as npm packages through GitHub Packages. To install or publish private skill packages, configure authentication using standard npm tooling.

## Quick setup

<Steps>

<Step title="Configure npm for GitHub Packages">

Use `npm login` or write your `.npmrc` directly:

```ini ~/.npmrc
@alavida-ai:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=ghp_your_token_here
```

</Step>

<Step title="Verify it works">

```bash
npm whoami --registry=https://npm.pkg.github.com
```

If this returns your GitHub username, you're ready to install private skill packages.

</Step>

</Steps>

## What to configure

| Item | Value |
|---|---|
| Scope | `@alavida-ai` (or your org scope) |
| Registry | `https://npm.pkg.github.com` |
| Token | GitHub personal access token |

The token is stored in your user-level `.npmrc` at `~/.npmrc`.

## Token requirements

Your GitHub personal access token needs these scopes:

| Scope | Purpose |
|---|---|
| `read:packages` | Install private skill packages |
| `write:packages` | Publish skill packages (authors only) |

Create a token at **GitHub > Settings > Developer settings > Personal access tokens**.

## .npmrc setup

Add these lines to your user-level `~/.npmrc`:

```ini
@alavida-ai:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=ghp_your_token_here
```

Or use an environment variable reference in a repo-level `.npmrc`:

```ini
@alavida-ai:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_PACKAGES_TOKEN}
```

Then set the variable in your shell:

```bash
export GITHUB_PACKAGES_TOKEN=ghp_your_token_here
```

## CI authentication

In CI, write the `.npmrc` from a secret:

```yaml .github/workflows/skills.yml
- name: Configure registry auth
  run: |
    echo "@alavida-ai:registry=https://npm.pkg.github.com" >> ~/.npmrc
    echo "//npm.pkg.github.com/:_authToken=${{ secrets.GITHUB_PACKAGES_TOKEN }}" >> ~/.npmrc

- name: Install skill packages
  run: npm install @alavida-ai/brand-copywriting
```

Or use the `setup-node` action with `registry-url`:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: 20
    registry-url: https://npm.pkg.github.com
    scope: '@alavida-ai'
  env:
    NODE_AUTH_TOKEN: ${{ secrets.GITHUB_PACKAGES_TOKEN }}
```

## Checking registry configuration

Use `npm config` to inspect your registry wiring:

```bash
npm config get @alavida-ai:registry
```
