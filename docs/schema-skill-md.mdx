---
title: "SKILL.md Reference"
description: "Complete reference for the SKILL.md file format — the authored skill artifact in agentpack."
---

`SKILL.md` is the skill artifact. It declares what the skill does, where its knowledge came from, what other skills it depends on, and what instructions agents should follow.

Every packaged skill directory contains exactly one `SKILL.md`.

## Frontmatter

The YAML frontmatter between `---` delimiters declares the skill's identity, provenance, and dependencies.

### Required fields

| Field | Type | Description |
|---|---|---|
| `name` | string | Skill identifier. Used for discovery directory names and display. |
| `description` | string | What the skill does. Should match `package.json` description. |

### Optional fields

| Field | Type | Description |
|---|---|---|
| `metadata.sources` | string[] | Repo-relative paths to knowledge files this skill was built from. |
| `metadata.status` | string | Lifecycle status: `"published"`, `"draft"`, or `"deprecated"`. |
| `metadata.replacement` | string | npm package name of the replacement skill (when status is `"deprecated"`). |
| `metadata.message` | string | Human-readable status note (e.g., migration guidance). |
| `requires` | object[] | Skill dependencies. Each entry has `name` and `package`. |

### Requires entries

Each item in the `requires` array declares a dependency on another skill package:

| Subfield | Type | Description |
|---|---|---|
| `name` | string | Human-readable name for the dependency. |
| `package` | string | npm package name (e.g., `"@acme/research"`). |

agentpack syncs `requires` entries into `package.json` dependencies automatically. You never edit `package.json` dependencies by hand for skills.

## Body

Everything after the closing `---` is the skill body. This is the markdown that agents read and follow at runtime. Write it as direct instructions.

The body can contain any valid markdown: headings, lists, code blocks, tables. Agents interpret it as capability instructions.

## Complete example

```markdown SKILL.md
---
name: brand-copywriting
description: Write brand-consistent copy using company tone and voice guidelines.
metadata:
  sources:
    - domains/brand/knowledge/tone-of-voice.md
    - domains/brand/knowledge/selling-points.md
  status: published
  message: Production-ready for all customer-facing copy.
requires:
  - name: editorial-principles
    package: "@acme/editorial-principles"
---

## Instructions

When writing copy for the brand, follow these guidelines:

1. Read the tone-of-voice principles before drafting.
2. Use active voice. Address the reader as "you."
3. Never use marketing superlatives.
4. Keep sentences varied in length for rhythm.

## When to use this skill

- Landing page copy
- Product descriptions
- Email sequences

## When NOT to use this skill

- Internal engineering documentation
- Legal or compliance content
```

## Minimal example

```markdown SKILL.md
---
name: hello-world
description: A minimal skill for testing agentpack.
---

Respond with a friendly greeting when the user says hello.
```

No `metadata`, no `requires`, no sources. This is the smallest valid SKILL.md.

## What SKILL.md owns

SKILL.md is the source of truth for:

- **Skill identity** — `name` and `description`
- **Source provenance** — which knowledge files the skill was built from
- **Dependencies** — which other skills this one requires
- **Lifecycle status** — whether the skill is published, draft, or deprecated
- **Agent instructions** — the body content agents follow

## What SKILL.md does NOT own

| Concern | Owned by |
|---|---|
| Package version | `package.json` (`version` field) |
| Publish configuration | `package.json` (`publishConfig` field) |
| Install state | `.agentpack/install.json` (runtime, per-repo) |
| Build hashes | `.agentpack/build-state.json` (generated by validate) |

<Note>
  `metadata.sources` paths are resolved relative to the repository root, not the skill directory. Run `agentpack skills validate` from the repo that contains those source files.
</Note>

<Warning>
  If you change `requires` in SKILL.md, run `agentpack skills validate` or `agentpack skills dev` to sync the change into `package.json` dependencies. The sync is automatic inside those commands — but it does not happen on its own.
</Warning>
